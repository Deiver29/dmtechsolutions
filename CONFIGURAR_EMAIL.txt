# CONFIGURACIÓN DE EMAIL EN XAMPP

## ⚠️ IMPORTANTE: Configurar para recibir emails

Para que el formulario envíe emails correctamente, debes configurar XAMPP para usar un servidor SMTP.

## 📧 OPCIÓN 1: Usar Gmail (Recomendado)

### Paso 1: Configurar php.ini

1. Abre el Panel de Control de XAMPP
2. Click en "Config" del módulo Apache
3. Selecciona "php.ini"
4. Busca la sección `[mail function]` y actualiza:

```ini
[mail function]
SMTP = smtp.gmail.com
smtp_port = 587
sendmail_from = techsolutionsdm00@gmail.com
sendmail_path = "\"C:\xampp\sendmail\sendmail.exe\" -t"
```

### Paso 2: Configurar sendmail.ini

1. Ve a: `C:\xampp\sendmail\`
2. Abre `sendmail.ini`
3. Actualiza estas líneas:

```ini
[sendmail]

smtp_server=smtp.gmail.com
smtp_port=587
error_logfile=error.log
debug_logfile=debug.log
auth_username=techsolutionsdm00@gmail.com
auth_password=TU_CONTRASEÑA_DE_APLICACION
force_sender=techsolutionsdm00@gmail.com
```

### Paso 3: Crear Contraseña de Aplicación en Gmail

1. Ve a tu cuenta de Google: https://myaccount.google.com/
2. Seguridad > Verificación en dos pasos (DEBES activarla)
3. Contraseñas de aplicaciones
4. Selecciona "Correo" y "Otro (nombre personalizado)"
5. Escribe "XAMPP DM Tech"
6. Copia la contraseña de 16 caracteres
7. Úsala en `auth_password` del sendmail.ini

### Paso 4: Reiniciar Apache

1. Detén Apache en XAMPP
2. Inícialo nuevamente
3. Prueba el formulario

## 🔍 SOLUCIÓN DE PROBLEMAS

### No recibo emails

1. **Verifica sendmail logs:**
   - Ve a: `C:\xampp\sendmail\`
   - Abre `error.log` y `debug.log`
   - Busca errores

2. **Verifica configuración Gmail:**
   - Contraseña de aplicación correcta
   - Verificación en 2 pasos activada
   - SMTP correcto (smtp.gmail.com:587)

3. **Verifica permisos:**
   - sendmail.exe debe tener permisos de ejecución
   - php.ini debe apuntar correctamente a sendmail.exe

### El formulario no se envía

1. Abre la consola del navegador (F12)
2. Busca errores en JavaScript
3. Verifica que `enviar_formulario.php` existe
4. Revisa la pestaña "Network" para ver la respuesta

### Emails van a spam

1. En Gmail, marca el email como "No es spam"
2. Agrega techsolutionsdm00@gmail.com a contactos
3. Considera usar un dominio propio en producción

## 🚀 PARA PRODUCCIÓN

En un servidor web real (NO XAMPP):

1. **Configura SPF, DKIM y DMARC** en tu dominio
2. **Usa un servicio profesional:**
   - SendGrid
   - Mailgun
   - Amazon SES
   - Postmark
3. **No uses mail() de PHP** - usa librerías como PHPMailer
4. **Implementa rate limiting** para evitar spam
